1. 核心目标
开发一个基于 Python 的轻量级游戏翻译辅助工具。通过全局快捷键触发，截取当前活动窗口的特定区域，进行 OCR 识别，并与预置的翻译文本库进行模糊匹配，最终在游戏画面上方以半透明置顶窗口形式显示翻译结果。

2. 功能需求 (Functional Requirements)
• FR-01 系统托盘驻留：
    ◦ 程序启动后最小化至系统托盘，不占用任务栏。
    ◦ 托盘右键菜单包含：设置、选择翻译库、重启服务、退出。
• FR-02 全局快捷键控制：
    ◦ 触发键（默认 Alt+Q）：执行“截图 -> 识别 -> 匹配 -> 显示”流程。
    ◦ 关闭/隐藏键（默认 Esc 或 Alt+W）：立即隐藏翻译窗口，但保持后台运行。
    ◦ 快捷键支持在设置中自定义。
• FR-03 智能截图与 OCR：
    ◦ 自动获取当前前台窗口（Active Window）的句柄和坐标。
    ◦ 调用 OCR 引擎（推荐 PaddleOCR 或 Tesseract）提取文本。
• FR-04 翻译匹配引擎：
    ◦ 核心逻辑：不依赖在线翻译 API（如 Google Translate），而是检索本地预存的 JSON/SQLite 数据库。
    ◦ 模糊匹配：考虑到 OCR 误差，必须使用模糊匹配算法（如 TheFuzz），设定相似度阈值（如 >85%）来查找对应的翻译文本。
• FR-05 富文本显示：
    ◦ 支持颜色、加粗、字体大小调整，以便区分人名、对话或系统提示。

3. 界面与交互需求 (UI/UX)
• UI-01 翻译悬浮窗 (Overlay)：
    ◦ 属性：无边框、背景半透明（透明度可调）、始终置顶（Always on Top）。
    ◦ 交互：支持鼠标拖拽移动位置、拖拽边缘调整大小。
    ◦ 穿透模式（进阶建议）：提供一个开关，锁定窗口位置后，鼠标点击可穿透翻译层直接操作游戏（Click-through），避免阻挡游戏操作。
• UI-02 状态反馈：
    ◦ 识别中显示 loading 动画或文字。
    ◦ 未找到匹配文本时显示友好提示。

4. 配置管理
• 配置文件存储在 config.json，包含：字体大小、背景透明度、快捷键映射、当前选用的翻译库路径。
